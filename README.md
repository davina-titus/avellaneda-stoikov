# Avellaneda-Stoikov Market Making Simulator

A full implementation of the canonical **Avellaneda & Stoikov (2008)** market making model , the closed-form solution to an optimal control problem for a limit order book market maker managing inventory risk.

## What this is

Market makers profit by capturing the bid-ask spread — but holding inventory is risky. If you buy at the bid and the price drops, you lose more than you earned in spread. The A-S model solves this formally: it uses **stochastic control** (the Hamilton-Jacobi-Bellman equation) to find the quotes that maximize expected utility over a finite horizon, explicitly trading off spread capture against inventory risk.

The result is elegant and closed-form. No simulation needed for the quotes themselves — only for the mid-price path and order arrivals.

## The math

### Setup

- Mid price follows arithmetic Brownian motion: `dS = σ dW`
- Market orders arrive as Poisson processes with intensity `λ±(δ) = A · exp(−k · δ±)`
  - Higher spread = fewer fills (exponential decay in k)
- MM maximizes expected CARA utility of terminal wealth: `E[−exp(−γ · W_T)]`

### Solution (HJB → closed form)

**Reservation price** (the MM's "fair value" given current inventory):
```
r(s, q, t) = s − q · γ · σ² · (T − t)
```
Inventory skews the reservation price — long inventory shifts quotes down to attract sellers, short inventory shifts up to attract buyers. This is the core insight.

**Optimal half-spread:**
```
δ* = γ · σ² · (T − t)  +  (2/γ) · ln(1 + γ/k)
```
Two components:
1. `γσ²(T−t)` — inventory risk term, shrinks as T approaches
2. `(2/γ)ln(1+γ/k)` — market impact / adverse selection term (constant)

**Optimal quotes:**
```
bid* = r − δ*/2
ask* = r + δ*/2
```

### Naive benchmark

Fixed symmetric spread posted at mid: `bid = S − δ*/2, ask = S + δ*/2`
No inventory awareness, no skewing. This is what a naive MM does.

## Results

Key findings across simulations:

- **A-S consistently outperforms naive** quoting in terms of Sharpe ratio and risk-adjusted PnL
- The edge is most pronounced at **high γ** (risk aversion matters) and **high σ** (volatile markets)
- **Inventory stays mean-reverting** under A-S; naive inventory drifts dangerously
- **Spread naturally narrows** as T → 0 — the MM becomes more aggressive near close to unwind inventory

## Usage

```bash
# Install dependencies
pip install -r requirements.txt

# Run interactive dashboard
python src/app.py
# → open http://127.0.0.1:8050

# CLI: single simulation with summary
python src/backtest.py

# CLI: parameter sensitivity sweep + export charts
python src/backtest.py --sweep --export

# CLI: Monte Carlo (500 paths)
python src/backtest.py --mc 500

# Custom parameters
python src/backtest.py --sigma 0.03 --gamma 0.2 --k 2.0 --mc 200 --export
```

## Parameters

| Parameter | Symbol | Default | Description |
|-----------|--------|---------|-------------|
| Volatility | σ | 0.02 | Asset volatility per unit time |
| Risk aversion | γ | 0.10 | MM's CARA risk aversion coefficient |
| Order intensity | k | 1.50 | LOB depth / fill rate decay |
| Arrival rate | A | 1.00 | Base rate of market orders |
| Horizon | T | 1.00 | Trading session length |
| Timestep | dt | 0.001 | Simulation resolution |
